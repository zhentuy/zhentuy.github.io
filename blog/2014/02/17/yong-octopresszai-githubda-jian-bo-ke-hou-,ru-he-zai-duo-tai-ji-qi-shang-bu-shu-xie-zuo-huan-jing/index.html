用octopress在github搭建博客后，如何在多台机器上部署“写作”环境

Feb 17th, 2014

在我窝里搭建好octopress的写作环境后，想在我工作场地也能随时写。 于是把本地博客repo的sourse分支push到github上，以为在公司pull下来就能用了。

在公司机器上要装好octopress后，把博客repo的sourse分支pull到本地，

rake new_post['test']

报错，说没有deploy 目录。 新建一个deploy 目录还是报错。回家后看到octopress的.gitignore 里面有deploy。 删除掉，push。到公司pull下来，继续rake，还是报错。

这时候想起来octopress的文档

Ask for and store your Github Pages repository url. Rename the remote pointing to imathis/octopress from ‘origin’ to ‘octopress’ Add your Github Pages repository as the default origin remote. Switch the active branch from master to source. Configure your blog’s url according to your repository. Setup a master branch in the _deploy directory for deployment.

deploy目录下有独立的git目录，里面的.git文件不会被push到github上。回家开VPN，把deploy目录下的.git scp 到工作用机器上，rake new_post[‘test’]成功
